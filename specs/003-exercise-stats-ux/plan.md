# Implementation Plan: Exercise Management, Statistics Polish & UX Feedback

**Branch**: `003-exercise-stats-ux` | **Date**: 2026-03-25 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `/specs/003-exercise-stats-ux/spec.md`

## Summary

Add exercise CRUD management and a pre-defined exercise seed to the Øvelser tab; improve statistics with IQR-based outlier filtering, a searchable exercise picker modal, and large-number formatting; add visual pressed-state feedback to long-pressable routine cards. No schema changes required — all new behaviour builds on the existing `pomp.exercises` table using ON CONFLICT for safe seeding and application-layer filtering for statistics polish.

## Technical Context

**Language/Version**: TypeScript 5.9.2 (strict mode)
**Primary Dependencies**: React Native 0.81.5, Expo SDK 54 (Expo Router 6), Supabase JS 2.x
**Storage**: Supabase PostgreSQL — schema `pomp` (no schema migration needed for this feature)
**Testing**: Jest (Expo preset) + `@testing-library/react-native`
**Target Platform**: iOS (iPhone 375–430pt); Android supported
**Project Type**: Mobile app (Expo Router file-based routing)
**Performance Goals**: Exercise list renders < 200ms; statistics outlier filter is purely in-memory (no extra DB round-trip)
**Constraints**: No new third-party dependencies — use Expo SDK and React Native built-ins only
**Scale/Scope**: ~50 seed exercises, single user per device; statistics charts handle up to ~500 sessions
